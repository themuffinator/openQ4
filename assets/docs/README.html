Open-source Quake 4 replacement

OpenQ4

A complete engine and game-code replacement for Quake 4, built to run stock assets on modern systems with updated rendering, display handling, input support, and build tooling.

Quick Start Highlights Compatibility Build Docs GitHub

Overview

OpenQ4 keeps official Quake 4 asset compatibility as its guiding constraint while modernizing rendering, display handling, input, packaging, and the build pipeline for current hardware and operating systems.

Single-player and multiplayer live under one baseoq4/ directory with game-sp and game-mp. SDK-derived game code is maintained in the companion OpenQ4-GameLibs repository, while BSE is treated as first-party source under src/bse/ and linked directly into the client executable.

Asset requirement: OpenQ4 does not include Quake 4 assets. You need a legitimate copy of Quake 4 from Steam or GOG to play.
Scope: OpenQ4 uses its own engine and game modules. It does not target compatibility with the proprietary Quake 4 DLLs and is not a drop-in runtime for legacy mods.

Highlights

Engine and compatibility

Stock content first

  • Complete engine and game-code replacement for Quake 4
  • Unified baseoq4/ game directory for SP and MP
  • Steam and GOG auto-discovery at startup
  • Official q4base media PK4 validation enabled by default, with retail game-binary PK4s ignored
  • In-tree BSE runtime integrated into OpenQ4-client_<arch>
Rendering and presentation

Modern image pipeline

  • Multi-scale bloom with luminance-based extraction
  • FP16 HDR scene targets, filmic tone mapping, and auto exposure
  • SSAO and optional CRT post-processing
  • Experimental shadow maps for projected and point lights, projected-light CSM, alpha-tested shadows, and optional translucent accumulation
  • Screen-fraction scaling, supersample-style presets, MSAA, and SMAA
Modern usability

Built for current systems

  • Automatic aspect-ratio, FOV, zoom, and view-weapon framing
  • Multi-monitor targeting plus borderless, desktop fullscreen, and exclusive fullscreen paths
  • Controller hotplug and analog input support
  • Meson builds with builddir/ for artifacts and .install/ for staged runtime packages
  • Windows, Linux, and macOS targets with x64 as the active baseline

Modern renderer upgrades layered onto the original game assets.

Quick Start

  1. Install Quake 4

    Use a legitimate copy from Steam or GOG.

  2. Download OpenQ4

    Grab the latest package from the Releases page.

  3. Extract the archive

    Unpack it anywhere you want to keep your OpenQ4 install.

  4. Launch the client

    Run OpenQ4-client_x64 or OpenQ4-client_x64.exe. OpenQ4 will try to locate your Quake 4 install automatically.

Runtime notes

Platform notes

  • Windows release packages are intended to be self-contained.
  • Linux currently uses an X11/GLX runtime path.
  • On Wayland, run through XWayland and make sure DISPLAY is available.
Manual path

If auto-detection fails

Set fs_basepath manually or check TECHNICAL.md for the full configuration reference.

OpenQ4-client_x64 +set fs_basepath "C:\Program Files (x86)\Steam\steamapps\common\Quake 4"

Compatibility and Scope

Already integrated

Compatibility work in place

  • BSE reconstruction and client-side effect execution
  • Effect-driven sound shader behavior
  • BSE screen and camera effect paths
  • Material and shader compatibility work needed to reduce custom override dependence
  • Modern display handling for widescreen, ultrawide, and multi-monitor setups
  • Official asset validation through fs_validateOfficialPaks 1
Project rules

Intentional boundaries

  • baseoq4/ remains the single game directory for SP and MP
  • BSE stays integrated in-tree under src/bse/
  • Dedicated server builds keep the disabled BSE manager path unless that changes by design
  • Canonical SDK/game-library edits belong in ../OpenQ4-GameLibs, not a mirrored src/game tree
  • Compatibility with proprietary Quake 4 game DLLs is a non-goal

Gameplay parity work is still ongoing. See TODO.md and docs-dev/release-completion.md for current follow-up items.

Building from Source

Full setup instructions live in BUILDING.md. The short version is to keep OpenQ4-GameLibs beside this repo at ../OpenQ4-GameLibs, use builddir/ for local Meson builds, and stage runtime packages into .install/.

Workflow

What matters

  • On Windows, use tools/build/meson_setup.ps1 instead of raw meson from an arbitrary shell.
  • Use meson install -C builddir --no-rebuild --skip-subprojects or the wrapper equivalent when staging .install/.
  • Set OPENQ4_BUILD_GAMELIBS=1 if you want the wrapper to trigger game-library builds during compile.
  • For local runtime validation, launch from .install/ so fs_cdpath points at the staged OpenQ4 overlay.
Windows example

Meson wrapper commands

powershell -ExecutionPolicy Bypass -File tools/build/meson_setup.ps1 setup --wipe builddir . --backend ninja --buildtype=debug --wrap-mode=forcefallback
powershell -ExecutionPolicy Bypass -File tools/build/meson_setup.ps1 compile -C builddir
powershell -ExecutionPolicy Bypass -File tools/build/meson_setup.ps1 install -C builddir --no-rebuild --skip-subprojects
